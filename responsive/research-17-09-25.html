Personal Notes

  • Runtimes: During the previous course, I already installed Node.js, so I am comfortable using tools which require it. However, I do not currently have other runtimes (like Ruby), and I prefer to avoid adding more dependencies to my setup. For this reason, I will prioritize standalone tools or those that work directly with Node, since they fit better with my current workflow and environment.
  • Bootstrap: Observing the documentation and examples, Bootstrap seems beginner-friendly and quick to implement due to its ready-made components. Customization might require extra effort to avoid a generic look.
  • TailwindCSS: Based on research, Tailwind looks very flexible and allows rapid prototyping. However, it requires learning a large number of utility class names, which could seriously slow me down.
  • Jekyll: Jekyll appears useful for static site generation, particularly for blogs and GitHub Pages. It seems to have a stable workflow but Ruby installation is a barrier.
  • Hugo: Hugo is extremely fast and easy to set up, running as a standalone executable without extra runtimes. Its simple templating system and clear structure make it efficient for building both small and large static sites with minimal configuration.
  • GSAP: GSAP seems to offer advanced animation capabilities and high flexibility for interactive projects. My observations indicate it might be overwhelming at first for those unfamiliar with complex timelines.
  • Anime.js: Anime.js looks similar to GSAP in terms of capabilities, but it seems to offer a simpler implementation and coding style, making it more approachable for small projects or lightweight animations.
  • React: React is a powerful JavaScript library for building dynamic user interfaces and single-page applications. Once set up, it enables reusable components, efficient rendering, and a flexible workflow for modern web projects.
  • Vite: Modern build tool, very fast, lightweight, and simplifies development workflow. Great for modern JS projects and experimenting with frameworks without heavy configuration.

Critical Reflection

For CSS frameworks, I prefer Bootstrap for quick prototyping, but TailwindCSS is great for highly customized designs. For animations, GSAP offers professional control, while Anime.js is suitable for small projects. Regarding static generators, Hugo is my choice for large sites due to speed, and Jekyll for simple blogs. The combination of these tools balances speed, flexibility, and minimal dependencies.
