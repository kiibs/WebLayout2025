Personal Notes

  • Bootstrap: Observing the documentation and examples, Bootstrap seems beginner-friendly and quick to implement due to its ready-made components. Customization might require extra effort to avoid a generic look.
  • TailwindCSS: Based on research, Tailwind looks very flexible and allows rapid prototyping. However, it requires learning a large number of utility class names, which could seriously slow me down.
  • Jekyll: Jekyll appears useful for static site generation, particularly for blogs and GitHub Pages. It seems to have a stable workflow but Ruby installation may be a barrier.
  • GSAP: GSAP seems to offer advanced animation capabilities and high flexibility for interactive projects. Observations indicate it might be overwhelming at first for those unfamiliar with complex timelines.
  • Anime.js: Anime.js looks similar to GSAP in terms of capabilities, but it seems to offer a simpler implementation and coding style, making it more approachable for small projects or lightweight animations.

My Preference

Based on my observations and research, I would choose GSAP and Anime.js for animations, and Bootstrap for CSS. GSAP provides professional-level animation tools, Anime.js offers a simpler implementation for lighter projects, and Bootstrap allows quick setup with prebuilt components. This combination seems versatile for building a dynamic and visually appealing portfolio.
